VISÃO GERAL · O QUE É O APP

Planejamento de refeições
com inteligência artificial

O ifood-android combina gerenciamento local de horários de refeição com recomendações de restaurantes via IA. O usuário define seus horários e preferências dietéticas — o app notifica 30 minutos antes de cada refeição e sugere onde pedir.

🕐

Agendamento de Refeições

4 refeições fixas com horários ajustáveis — Café, Almoço, Lanche e Jantar — persistidos em Room.

🥗

Preferências Dietéticas

Tags de preferência por tipo de refeição. Swipe para deletar. Filtragem por MealType via Room.

✨

Recomendações por IA

Backend Node.js consome preferências do usuário e retorna restaurante, prato, preço e endereço.

UX · JORNADA DO USUÁRIO

Do primeiro acesso
à recomendação

SplashViewModel determina a rota inicial checando a flag de onboarding no DataStore e a permissão POST_NOTIFICATIONS.

Onboarding (4 páginas): Pager com apresentação do app. Dados do usuário coletados. Flag salva no DataStore — executado uma única vez.
Permissão de Notificação: Solicitação de POST_NOTIFICATIONS no Android 13+. Tela de fallback se negada.
Home Screen: Exibe horários das 4 refeições, preferências ativas e acesso à recomendação de IA. Defaults semeados na primeira execução via SeedDefaultSchedules.
Ajuste de Horário: ScheduleAdjustmentScreen permite editar hora e minuto de cada refeição. Alarme reagendado automaticamente.
Notificação Proativa: 30 minutos antes da refeição, o AlarmManager dispara → WorkManager consulta a API → notificação com recomendação exibida.

FEATURES · AGENDAMENTO & PREFERÊNCIAS

Controle total
sobre as refeições

Quatro slots fixos com horários padrão inteligentes. Alarmes reagendados a cada alteração. Preferências por tipo de refeição com persistência imediata.

🗓️

Agendamento de Refeições

Quatro slots fixos: Café 08h · Almoço 13h · Lanche 17h · Jantar 21h. Defaults semeados por SeedDefaultSchedules na primeira execução. UpdateMealSchedule persiste e reagenda o alarme imediatamente.

🥗

Preferências Dietéticas

Tags de preferência associadas a um ou mais MealType. Armazenadas como CSV no Room (mealTypes). Filtradas por tipo com GetPreferencesByMealType. Swipe-to-delete com confirmação visual.

🔔

Ciclo Diário de Alarmes

Após o disparo e envio da notificação, o alarme é automaticamente reagendado para o dia seguinte. BootReceiver escuta BOOT_COMPLETED e restaura todos os alarmes após reinicialização do dispositivo — sem intervenção do usuário.

Por que AlarmManager?

Precisão de disparo no horário exato, mesmo com o app em background. setExactAndAllowWhileIdle para contornar Doze Mode.

Por que WorkManager?

Trabalho garantido mesmo com restrições de rede ou bateria. Integração nativa com Hilt via HiltWorker. Retry automático em falha.

FEATURES · RECOMENDAÇÕES POR IA

Recomendação personalizada
para cada refeição

O backend Node.js/TypeScript recebe o contexto do usuário e retorna uma sugestão completa de restaurante. A recomendação é exibida num BottomSheet interativo na HomeScreen.

// Request → Backend API data class MealRecommendationRequest( val userName: String, val mealType: String, // "LUNCH" val preferences: List<String> // ["vegetariano", "sem glúten"] ) // Response ← Backend API data class MealRecommendationResponse( val restaurantName: String, val dish: String, val price: Double, val address: String )

Contexto Rico

Nome do usuário + tipo de refeição + lista de preferências dietéticas ativas para aquele horário.

Backend Separado

ifood-backend/ é um projeto Node.js/TypeScript independente, não incluído no build Android.

UI não bloqueante

BottomSheet com estado de loading/success/error. ViewModel expõe via StateFlow; coroutine no IO dispatcher.

INFRAESTRUTURA · CI/CD

Release — push para master

assembleRelease + bundleRelease — assinatura com keystore via secrets
gh release create — tag v{versionName}, APK + AAB anexados automaticamente
